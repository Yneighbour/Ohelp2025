# 智慧养老系统 - 前端（Vue 3 + Vite，JS）

本目录已从“纯静态页面演示”升级为 **Vue 3 + Vite（纯 JS）** 的前端工程，并已接入后端 `o-b` 的核心 REST API（认证、健康、活动、紧急呼救、服务预约、部分管理端操作等）。

## 📱 功能页面

- 手机号和密码登录
- 记住密码功能
- 注册链接提示

- 点击卡片查看详情

### 7. 社交动态列表页面

### 8. 养老活动列表页面
- 活动分类筛选

### 9. 活动详情页面
- 活动横幅图片

- 活动介绍和注意事项

### 10. 个人中心页面
- 用户信息展示

### 11. 管理端后台页面（/admin-*）
- 用户列表：支持筛选 + 启用/禁用/删除（已调用后端接口）
- 角色管理：角色列表与权限配置原型

- 老人档案：老人列表与健康状态标签（读接口已接入）
- 健康记录：全量健康记录列表（读接口已接入）
- 家属绑定：老人-家属关联视图（读接口已接入）
- 服务项目：服务项目列表演示

- 活动列表：活动列表查看，支持取消活动（调用活动下架接口）
- 报名管理：活动报名管理原型页

## 🚀 快速开始

### 1) 启动后端（o-b）
后端默认地址：`http://localhost:8080/api`

### 2) 启动前端（o-f）
在本目录执行：

```bash
npm install
npm run dev
```

默认前端地址：`http://localhost:5173`

登录说明：
- 使用预置管理员账号登录（见根目录 README 中“初始化数据”），在登录页选择“管理端”角色，可直接进入后台 `/admin-user-list`；
- 选择“用户端”角色则进入健康首页 `/health`，可通过底部导航访问其他用户侧页面。

### API 地址与代理
- 默认使用 `VITE_API_BASE_URL=/api`（见 `.env.example`）
- 开发期通过 `vite.config.js` 的 proxy 将 `/api` 代理到 `http://localhost:8080`

## 🎨 设计特点

### 适合老年人的设计
- **大字体**：基础字体16px，重要信息18px以上
- **大按钮**：所有可点击元素最小44x44px
- **清晰布局**：简洁明了，避免信息过载
- **颜色对比**：符合WCAG AA标准，对比度≥4.5:1
- **直观图标**：配合文字说明，易于理解

### 响应式设计
- 移动设备优先（< 768px）
- 平板设备适配（768px - 1024px）
- 桌面设备支持（> 1024px）

### 技术特点
- 基于 Vue 3 + Vue Router + Vite 的单页应用
- 样式与旧版静态页面共用 `styles/*.css`，保证视觉一致
- 业务逻辑与路由已迁移为 Vue 组件（见 `src/views` 与 `src/router`）
- 普通列表/卡片页仍然是纯前端渲染，健康详情趋势图继续使用 Canvas 绘制
- 旧版 `scripts/` 下的纯原生 JS 仅保留为设计/对照参考，不再作为入口

## 🔗 前后端对接概况（简要）

> 详细的接口与页面对应关系见 `前后端交互对接说明.md`，这里只做简要概括。

- 已接入后端的核心交互：
  - 登录与 Token 校验
  - 健康记录读取（用户端健康视图 + 管理端健康记录）
  - 活动列表与详情读取
  - 紧急呼救上报与历史记录查询
  - 服务预约列表读取 + 取消/完成操作
  - 管理端用户列表读取 + 启用/禁用/删除
  - 管理端老人档案与家属绑定列表读取
- 仍为前端演示的数据：
  - 社交圈、活动报名记录、部分个人中心设置/联系人管理
  - 管理端角色管理、权限配置、服务项目与报名管理等部分操作

## 📝 注意事项

- 后端接口统一返回包裹：`{ code, message, data? }`；前端在 `src/api/http.js` 做了统一解包与错误处理。
- 登录流程：`/auth/login` 返回自定义 token，前端按 `data.token` 是否存在判断成功与否，并将 token 与角色等信息存入 `localStorage`；进入受保护路由前会调用 `/auth/validate/{token}` 做一次“软校验”。
- 管理端 `/admin-*` 路由仅在本地角色为 `admin` 时可访问，且在后台顶部提供“退出登录”按钮，一键清理会话并返回登录页。
